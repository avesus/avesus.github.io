What The Paper Gives

The paper defines a minimal computation model whose primitives are ordinary Boolean hardware primitives: wires, constants, and 2:1 multiplexers. With feedback and registers, those primitives become sequential switching networks. With configuration registers, the same substrate can be programmed. With unbounded tiling, the construction has unbounded memory and therefore universal computational power.

The point is implementation-level sufficiency. Conditional selection, state, reconfiguration, routing discipline, and replication can all be expressed inside the same Boolean system.

Why MUXes Are Enough

Constants supply fixed zeros and ones. Wires move bits. A 2:1 MUX selects one of two inputs under control of a select bit. Networks of those MUXes implement arbitrary Boolean functions.

Once outputs feed registers and registers feed later logic, the fabric becomes a clocked state machine. Once some registers are treated as configuration, structure becomes data. Once the pattern can be tiled without a fixed global bound, memory and machine size are no longer fixed by the first drawing.

Construction Recipe

  1. Start with wires and constants so bits can be moved and fixed values can enter the network.
  2. Use 2:1 MUXes as the switching primitive. A MUX selects one input under control of one bit.
  3. Compose MUXes into arbitrary Boolean functions.
  4. Add feedback through registers so the network has clocked state.
  5. Reserve some state as configuration registers so the next-state function can be changed by data.
  6. Tile the same local pattern without a fixed bound so memory and machine size can grow.

That chain is the whole claim. It does not require a hidden CPU, a special instruction set, or an external programming oracle. The same kind of Boolean state that computes values can also carry the configuration that changes the fabric.

Implementation Path

Read this as an implementation model for a reconfigurable substrate. The important terms are local wires, constants, 2:1 MUXes, feedback, configuration state, tiling, and the protocol that installs new configuration.

Reconfiguration is not outside the machine. Programming a fabric is itself a Boolean, clocked process. Configuration bits move through a substrate, land in registers, and change the next-state function the same way any other state changes it.

Construction Boundary

The construction is intentionally small. It does not start from a convenient CPU, an HDL standard library, or a friendly programming language. It starts from the fewest Boolean pieces that still reach state, configuration, and unbounded growth.

The first description is lower-level than the machines people usually name. The gain is that every layer has a direct substrate interpretation.

Cartilage Visual Language

The 32 Cartilage cell-role codes are a compact body-type alphabet: reconfiguration port, cross, constants, four wire orientations, and six MUX modes in four orientations. The Cartilage Visual Language article shows the rendered key and decodes all 32 codes.
