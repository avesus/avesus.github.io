What changed in this revision

The claim is now stated at the level of computational primitives: every machine-visible value and transformation remains Boolean, while locality, persistence, matter, energy, and finite propagation are explicit physical premises rather than consequences of an algebraic identity.

  • Configuration is Boolean state interpreted and installed by fixed Boolean circuitry inside the same fabric.
  • CLOCK is a local, potentially aperiodic transaction strobe—not a global clock domain or timebase.
  • Finite replacement now has an explicit ownership-tree protocol for staging, traversal, commit, acknowledgement, release, and application fencing.
  • Universality is exactly Turing equivalence under constructive execution and potential-extension premises; unbounded passive memory alone is not enough.
  • The seven-bit result is minimal only for the selected 32-role, four-direction direct encoding. The current browser and RTL package proves one exact local installation while leaving the stronger transaction and physical-realization obligations explicit.

Abstract

This paper gives a finite local construction in which computation, storage, routing, configuration, and reconfiguration are performed by one kind of machinery: Boolean switching circuits. The substrate is a homogeneous two-dimensional fabric of identical cells. Every cell is a fixed Boolean sequential circuit. A finite Boolean configuration word selects the cell's effective role as a wire, constant, crossing, 2:1 multiplexer, or local reconfiguration port. Further Boolean state records local ownership and implements the configuration loader. A configuration is therefore not metadata interpreted by a machine outside the model. It is state inside the same Boolean machine whose behavior it selects.

An owning region configures a finite child region by presenting Boolean DATA and CLOCK signals to the child's local reconfiguration port. Fixed Boolean circuitry in the cells routes that stream through the child's ownership tree, records progress, installs cell records, and reports completion. CLOCK is neither global nor continuously broadcast: loading follows one selected branch, while the final commit strobe is fanned only through the finite target region and need not arrive simultaneously. It is an application signal produced by the owner. Under a bounded-delay discipline, the owner need only separate its CLOCK transitions by a finite interval determined by the target region's maximum causal depth. A delay path made from the same cells can provide that interval. No global clock, global or local H-tree, continuously running clock-distribution network, local crystal oscillator, GPS reference, or non-Boolean configuration controller is required. The finite transaction tree does distribute a finite strobe; it is not required to be balanced or zero-skew.

The construction is closed under reconfiguration: a configured Boolean region can compute a configuration stream which causes another region to become a new Boolean circuit, including a circuit containing further reconfiguration ports. Finite-state machines, conventional stored-program processors, tapes, stacks, and linked storage are finite configurations or potentially unbounded organizations of this one substrate. With finitely described initial conditions and arbitrarily extensible finite active regions, the model is Turing-equivalent. The contribution is not a new computability class. It is an explicit demonstration that no change of computational ontology is required between Boolean logic, state, programming, and the machines produced by programming.

1. The claim

The statement "Boolean algebra is all that is required" is a claim about computational primitives.

It means:

  1. Every value carried by the substrate is Boolean.
  2. Every local combinational transformation is a Boolean function.
  3. Every stored configuration, ownership indication, progress indication, and application value is Boolean state.
  4. Every change to that state is caused by Boolean signals acting through Boolean switching circuitry.
  5. A circuit which installs or replaces another circuit is not a second-order interpreter outside the substrate. It is another Boolean circuit inside the substrate.

The claim does not assert that the equations of Boolean algebra manufacture matter, energy, spatial adjacency, or propagation delay. Any physical computer requires a realization. Cartilage requires only that its realization provide stable Boolean levels, state-holding switching structures, local connections, and finite causal propagation. These are properties of a physical Boolean circuit, not additional computational data types or abstract machines.

In this paper, a Boolean process means a retained network of Boolean operations and Boolean storage connected by causal signal paths. Two processes may compute the same extensional truth function while differing in placement, propagation depth, feedback, or ownership. Those differences remain part of the machine even though they are erased by ordinary truth-table equivalence.

The distinction is important. A truth table describes a Boolean function after the structure and timing of its realization have been discarded. Cartilage does not discard them. Its primitive object is a locally connected Boolean switching process whose effective connections and functions are themselves selected by Boolean state.

George Boole's calculus supplies the algebra of Boolean operations and predicates. His writings also discuss successive operations and propositions interpreted over portions of time, but the formal calculus does not prescribe gate delay or a next-state schedule. This paper does not attribute such a schedule to an algebraic identity. It retains Boole's value algebra while refusing to erase the causal switching structure which realizes Boolean operations. The additional premises are explicit physical locality, persistence, and propagation—not another computational value or transformation algebra.

2. The fixed substrate and the circuits it realizes

2.1 Carrier fabric

Let

L=ℤ2

be a regular lattice. A site has four fixed physical neighbors: left, up, right, and down. The use of an infinite lattice is a concise representation of potential extension. Every configuration, region, input, transaction, and finite-event causal cone considered below is finite or finitely described. No result depends on an actually fabricated infinite device.

For the computability theorem, L is initialized by one computable rule: a finite seed is allocated and every other site contains the same unclaimed quiescent cell state. It does not contain an arbitrary infinite program. At every finite event prefix, non-Zeno local propagation permits only a finite claimed causal region. A family of finite physical devices can realize successively larger finite prefixes of this abstract machine; physical attachment is an implementation resource action, not silently identified with the abstract CLAIM operation defined in Section 7.4.

Each site contains the same fixed cell circuit. The fixed carrier graph is denoted by G0. It consists only of the repeated cells and their nearest-neighbor physical links.

2.2 Cell state

For a site v, let its stored Boolean state be

zv=(Av,Cv,Pv,Qv,Bv).

The fields have the following meanings.

  • Av indicates whether the site has been claimed into the active finite fabric.
  • Cv is the installed role code.
  • Pv is the ownership or parent-direction code.
  • Qv is application signal or retained application state.
  • Bv is the fixed loader's working state: configuration shift bits, progress, completion, and edge-history bits.

The exact number of working bits is an implementation choice. It does not change the argument, provided it is finite and every working transition is Boolean. The concrete Cartilage encoding uses a seven-bit installed record, consisting of five role bits and two parent-direction bits. A complete robust loader additionally retains a seven-bit shadow record, a finite bit index, BUSY and FULL indications, the preceding CLOCK value, and the old parent/root relation used by the active transaction. Newly received role or parent bits therefore cannot cut the route still being used to finish the transaction.

The cell has a fixed local transition circuit

F:{0,1}k+4m→{0,1}k+4m,

where the arguments are the cell's retained bits and the Boolean signals presented by its four neighbors. This notation names the fixed cell circuit; it does not impose a global update clock. In an event-driven realization, each output transition of F occurs after the causal delay of the gates and links involved.

2.3 Installed role alphabet

The five-bit role code selects one of 32 effective roles:

  • 24 oriented 2:1 multiplexer roles;
  • four oriented wire roles;
  • one crossing role;
  • one forced-zero role;
  • one forced-one role;
  • one reconfiguration-port role.

The 24 multiplexer roles arise from four rotations and six ordered choices of two data inputs from the three non-selector sides. A normal cell broadcasts its selected output toward its neighbors. The crossing role carries the two opposite routes independently. The crossing is a routing convenience; it adds no new Boolean function.

The reconfiguration port is not a different kind of hardware. It is one value of the same five-bit role field. Installing a port therefore requires the same operation as installing a wire or a multiplexer.

2.4 Two graph levels, one computational ontology

The fixed carrier graph G0 must be distinguished from two decoded graphs:

G(A,C,P)=the effective application, port-ingress, and routing graph,

and

T(A,C,P)=the ownership forest decoded from claimed cells, roles, and directions.

Unclaimed sites are absent from both decoded graphs. In T(A,C,P), a claimed cell whose role is a reconfiguration port is a component root; its two direction bits select its external ingress orientation rather than an internal parent edge. Every claimed non-port cell contributes the locally directed parent edge selected by its two direction bits. Thus root status cannot be decoded from P alone. Likewise, the effective behavior of a port in G depends on both its role and its ingress orientation.

The carrier graph is fabricated. The effective circuit graph and ownership graph are not supplied by an external interpreter. They are decoded locally by the fixed Boolean cell circuits from Boolean state. When a configuration stream changes C or P, the same fixed Boolean substrate changes the effective graphs it presents.

This is the same logical recursion used by an SRAM FPGA. A fabricated FPGA is already a Boolean sequential circuit containing configuration storage, routing selectors, lookup structures, registers, and loading circuitry. If U denotes that fixed circuit and C its configuration state, then

U(C,x)=fC(x).

The bit pattern C specializes the fixed circuit into the effective circuit fC. Cartilage makes this specialization local and recursively available through owned ports.

3. Boolean completeness, storage, and causal time

3.1 Multiplexer completeness

A 2:1 multiplexer computes

M(s,a,b)=(¬s∧a)∨(s∧b).

With constants zero and one,

¬x=M(x,1,0),

and every finite Boolean function can be constructed recursively by Shannon expansion:

f(x,y)=M(x,f(0,y),f(1,y)).

The earlier MUX Algebra working note develops the two-variable case in full: nested selection notation, a four-bit configuration, and all 16 Boolean functions of two inputs.

Consequently, the cell's fixed loader, counters, decoders, progress flags, ownership tests, routing selectors, and application logic can all be reduced to multiplexers, constants, and stored Boolean values. The interpretation, movement, and installation of configuration bits require no richer logical primitive than the circuits being installed.

3.2 Storage

A retained bit can be constructed from a configured MUX and a positive-delay feedback route. Connect the MUX's first data input to its own prior output through that route, connect its second data input to new value d, and use enable e as selector. Its Boolean next-value relation is

q+=M(e,q,d).

When e=0, the delayed prior value q is selected and retained. When e=1, d is selected and becomes the value subsequently presented through the feedback path. Holding e and d for the declared setup, propagation, and hold intervals gives an addressable one-bit latch. A physical latch, flip-flop, charge-holding node, or other bistable switching structure realizes the same relation causally. The stored value is Boolean; the load, hold, and feedback paths are Boolean circuits. An initial value is part of the finite initial configuration.

Purely equational Boolean algebra does not specify how long a transition takes. Cartilage makes no such claim. It requires a causal realization of the Boolean circuit. The realization is non-Zeno: it cannot perform infinitely many causally successive local transitions in a finite physical interval. Once finite propagation is retained rather than abstracted away, feedback provides retention and transitions provide an ordering of events without requiring a universal time coordinate.

3.3 No operational global state

The behavior of one cell depends only on its own retained bits and signals arriving from adjacent cells. Two events with no causal path between them need not be ordered. A mathematical observer may describe values on a chosen cut through the event history, but the machine does not read, construct, or update a simultaneous global state.

The browser emulator evaluates the lattice through repeated framebuffer passes. Those passes are an integration and scheduling device of the emulator. They impose one simulated propagation increment per pass; they are not a clock distributed to the modeled Cartilage machine.

4. Finite regions, ownership, and ports

4.1 Valid owned region

A finite owned region is one current component of the decoded ownership forest, written

R=(VR,pR,TR),

where:

  • VR⊂L is a finite connected set of claimed cells;
  • pR∈VR is the only cell in VR whose installed role is reconfiguration port;
  • every other cell in VR is a non-port cell with one parent direction pointing to an adjacent cell in VR;
  • following parent directions from every cell reaches pR after finitely many steps;
  • no parent-direction cycle exists inside R;
  • VR is closed for this transaction: no cell outside VR is accepted by a member as a child of the retained transaction tree.

Thus TR is a rooted spanning tree of exactly one ownership component. A nested child port is the root of a different component and is not traversed as an ordinary member of VR; that component is a transaction-closed boundary unless separately selected through its own port. The port belongs to the child region and is its root. Its parent/ingress code selects one local application side for DATA; the next application side in the fixed rotational convention supplies CLOCK. These are two distinct ordinary Boolean signal paths driven by one logical owner. The port's separate configuration-overlay channels participate in the child's ownership tree. The single-driver condition excludes configuration-ingress contention.

One convenient numeric decoding is 0=left, 1=up, 2=right, and 3=down. The port root uses the same two bits to identify its external DATA ingress orientation; its CLOCK ingress is the conventionally adjacent side and its root status is supplied by the port role itself.

Let

D(R)=maxv∈VRdistTR(pR,v)

be the ownership-tree depth. Because R is finite, D(R) is finite.

4.2 Ownership is authority, not address space

Under the valid-tree and exclusive-driver contract, the parent field serves two related purposes:

  1. it supplies the local route by which configuration control reaches a cell; and
  2. it identifies the unique enclosing port whose well-formed transaction may replace that cell.

No global coordinate is needed during a transaction. The owner addresses the child by possessing a physical signal path to the child's port. The port and the tree behind it define the scope of the operation.

The ownership tree is not an application heap. It is the configuration-authority structure. An application heap is an encoded graph of objects and references implemented by ordinary configured storage and routes. It may contain sharing and cycles even though the physical ownership relation remains a tree.

4.3 Owner and loader responsibilities

The owning application region determines:

  • the sequence of seven-bit cell records to install;
  • the Boolean DATA waveform carrying those records;
  • the Boolean CLOCK waveform delimiting record bits;
  • a safe interval between CLOCK transitions for the particular child region.

Before replacement, the owner poisons, fences, or otherwise isolates the child's application boundary and prevents ordinary application traffic from entering until release. The base theorem guarantees a correct final replacement. It does not claim useful hazard-free application behavior from the intermediate commit wave.

The fixed loader circuitry distributed through the child determines:

  • which incomplete cell receives the current bit;
  • how CLOCK and DATA are forwarded one neighbor at a time;
  • when a cell record is complete;
  • when a subtree is complete;
  • when the whole region is complete.

The owner may contain a processor, finite controller, dataflow network, or dedicated bitstream generator. It is itself a configured region of the same fabric. The loader is not software executed by a hidden processor. It is fixed Boolean circuitry in every cell.

5. The finite reconfiguration transaction

5.1 Installed record

For the base role alphabet, one installed cell record is

wv=(Cv,Pv)∈{0,1}7.

Five bits select one of 32 roles and two bits select one of four parent directions. Optional application-state initialization or prototype-inheritance fields may be added, but they are not required for Boolean closure or universal computation.

CLOCK edge recognition is itself Boolean. If c is the current CLOCK input and c− its retained preceding value, then

rise=¬c−∧c,fall=c−∧¬c,(c−)+=c.

The bit index, FULL transition, commit enable, and reset/release conditions are finite Boolean functions of these edge indications and the loader's retained bits.

5.2 Deterministic traversal

Each cell orders its possible child directions relative to its parent direction. Any fixed order is sufficient. The concrete Cartilage order used here is right-relative, forward-relative, then left-relative.

The loader applies the following rule locally:

  1. If an ordered child exists and is not complete, forward CLOCK and DATA to the first such child.
  2. If every child is complete, consume incoming DATA bits into the local staging register.
  3. When the local record is complete, mark the cell complete toward its parent.

In COMMIT state, a complete node forwards the commit-high transition to every retained transaction child. A leaf then copies its shadow record; an internal node copies only after every retained transaction child has acknowledged commit. Commit-low and release likewise use the retained transaction route. No new active role or parent value controls the transaction which installed it.

This is a depth-first postorder traversal generated without a traversal stack. The ownership pointers are the stack embedded in space, while each completion bit records which subtrees no longer require data.

On the first transaction event observed by a cell, the loader retains its current parent direction and whether it is the transaction root. These retained bits define the transaction route. Loading, completion, commit dissemination, and release use this route rather than newly received role or parent bits. Incoming records remain in shadow latches while loading proceeds.

After every cell is complete, one additional commit cycle is forwarded through the complete transaction tree. It causes a postorder copy of shadow role and parent records into active latches: leaves copy first, and each internal node copies after its children acknowledge. The cells continue routing commit and its guarded return phase using the retained transaction route. A postorder release then clears transaction working state.

For the base theorem, a valid final ownership forest partitions every cell of R into disjoint rooted components. Every installed port is exactly one component root; no non-root is a port. Every non-root parent direction points to an adjacent cell in the same component, and repeated parent following terminates at that component's port. The original root pR remains a port, preserving the enclosing owner's interface.

To denote an immediately usable nested hierarchy, the final configuration must satisfy an additional local wiring condition. For every new port root, the DATA and CLOCK driver sites selected by its ingress orientation must exist, lie in one different component designated as its immediate owner, and be configured to drive that port. Directing an edge from that owner component to the child component must produce an acyclic component graph rooted, within R, at the component containing pR. Under this condition each additional component is a genuine locally driven child. A forest which lacks the driver wiring remains a valid partition, but its extra ports are not thereby claimed to be usable. Deletion or return of capacity to an enclosing region is a separate allocation operation, not part of the base replacement theorem.

This finite overlay epoch permits a transaction to install new ports and new parent directions without allowing those new values to redirect their own installation. It requires only Boolean latches, completion bits, and multiplexers in the fixed loader.

5.3 Transaction modes and termination

Each loader has a finite Boolean mode field encoding the following protocol states:

  • IDLE: no transaction route is retained and FULL is clear.
  • LOAD: the transaction route is retained; the first incomplete frontier consumes DATA on rising strobes.
  • FULL: the local shadow record and every transaction-child subtree are complete; FULL remains asserted toward the transaction parent.
  • COMMIT: a commit-high event is being forwarded through every FULL transaction child.
  • COMMITTED: the local active record has been copied from shadow and commit acknowledgement has been returned toward the root.
  • RELEASE: the commit-low/release event is clearing FULL, shadow-valid, acknowledgement, and transaction-route state in a safe postorder.

The first loading strobe entering an IDLE region establishes BUSY/LOAD and causes each visited cell to retain its current transaction route. Seven rising strobes at a selected cell fill its shadow record; the following guarded low phase makes that cell FULL. FULL is monotone until release.

After the root observes every transaction child FULL and becomes FULL itself, the next rising strobe is unambiguously COMMIT rather than an eighth data strobe. A COMMIT node first asserts commit-high on every retained transaction-child output. A leaf then copies shadow to active state and acknowledges its transaction parent. An internal node waits until every transaction child acknowledges, then copies its own shadow record and acknowledges. Root acknowledgement is COMPLETE.

The owner keeps the application boundary fenced, holds the commit phase for its declared causal guard, and then lowers the strobe. RELEASE propagates through the retained tree. A releasing node asserts RELEASE_ACK on its retained parent edge and holds both that ACK and the retained route for a characterized one-hop capture guard long enough for the parent to latch it. An internal node begins its own capture guard only after latching every child RELEASE_ACK. It clears transaction state after that guard; consequently the root clears last. The complete release path, including these capture guards, is included in ΛL(R). COMPLETE then returns low, the retained transaction overlay is gone, and the newly installed forest becomes the only ownership decoding used by later transactions.

All modes, child tests, monotone flags, fan-out enables, acknowledgements, copies, and clearing conditions are finite Boolean state and Boolean functions.

5.4 Traversal lemma

Lemma 1 — unique finite coverage. For every valid finite owned region R, the local traversal rule selects every cell exactly once and terminates at the root.

Proof. A one-cell tree has no children, so its root consumes one record and completes. Assume the statement for trees smaller than n. In a tree of n cells, the root forwards the stream to its first incomplete child. That child's subtree is smaller than n, so by induction it consumes exactly one record per cell and completes. The root then selects the next incomplete child. The child order is fixed and a completed child is never selected again. After all child subtrees complete, the root consumes exactly one record and completes. Every non-root node belongs to exactly one child subtree because the ownership relation is a tree. The retained transaction route is unaffected by shadow or newly committed records, so the induction remains valid while roles and parent directions are replaced. Therefore no node is omitted or repeated. ∎

The owner supplies records in the postorder induced by the transaction tree. Within each record it supplies five role bits—two orientation bits and three mode bits—and two parent bits in their declared least-significant-bit-first order. For N=|VR|, loading consumes exactly 7N rising strobes. One additional guarded cycle disseminates commit. Completion and release carry no additional cell record.

5.5 Installation lemma

Lemma 2 — local Boolean staging and installation. If a selected cell receives seven DATA bits separated by valid CLOCK transitions and subsequently receives commit, then its installed record becomes exactly those seven bits using only Boolean state transitions.

Proof. The fixed loader shifts one Boolean DATA value into a seven-bit shadow register on each designated CLOCK event. A finite Boolean counter or equivalent shift-completion signal recognizes the seventh received bit and makes the cell complete for loading purposes. The later commit event enables copying from the shadow register into the active role and parent latches. Shift, count, equality detection, completion, enable, and copying are all finite Boolean functions with retained Boolean state. ∎

5.6 Commit-and-release lemma

Lemma 3 — finite commit and safe release. For every finite transaction tree whose cells are FULL, one commit-high event causes every cell to copy its shadow record exactly once and causes COMPLETE at the root. The subsequent release event clears every retained transaction route exactly once without cutting a still-required commit, acknowledgement, or release path.

Proof. A FULL leaf has no transaction child. It copies once, latches COMMITTED, and acknowledges. Assume the result for child subtrees of height less than d. A node at height d forwards commit to all children. By induction every child subtree commits and acknowledges in finite causal depth. The node then copies once and acknowledges. Thus the root eventually receives all acknowledgements and asserts COMPLETE. For release, a leaf asserts RELEASE_ACK and retains its route for the one-hop capture guard, so its parent necessarily latches the acknowledgement before the leaf clears. Inductively, an internal node latches every child acknowledgement, asserts its own, and retains its route for the same guaranteed capture before clearing. The root clears last. No active or shadow parent value participates in this proof; all paths are retained transaction paths. ∎

5.7 Endogenous-reconfiguration theorem

Theorem 1 — Boolean closure under finite reconfiguration. Let R be a valid finite owned region with one designated configuration driver and a quiescent application boundary. Let an owning region emit a finite valid configuration transaction at pR, and let the new role and parent records jointly decode to a valid final ownership forest. Then the transaction installs the specified Boolean record in every cell of R, changes no installed record outside R, terminates, releases a well-defined final circuit, and invokes no computational mechanism outside the Boolean fabric.

Proof. By Lemma 1, the fixed local traversal delivers one record to every cell and terminates. Transaction closure and child-only forwarding prevent the stream from selecting a cell outside R. By Lemma 2, each record is staged by Boolean sequential circuitry. Lemma 3 commits and releases the finite tree without severing a still-required path. Commit is causally progressive rather than simultaneous, but the application fence hides the intermediate circuit from external application use. After root acknowledgement and release, the stipulated final records jointly decode to the valid final forest. The port is an installed Boolean role. Routing decisions are functions of Boolean transaction-route and completion bits. Configuration transport uses Boolean signals. The installed application and ownership fields are Boolean state. The owner is assumed here to emit finite Boolean DATA and CLOCK waveforms; Section 7.3 shows that such a generator is itself a finite Cartilage configuration. Therefore production, transport, traversal, progress recognition, commit, release, and the resulting circuit are all processes of the same fixed Boolean substrate. ∎

When the final driver-wiring and component-acyclicity condition of Section 5.2 holds, a newly installed port may receive another locally produced configuration transaction and instantiate its own finite child region. Recursion introduces no new kind of machine.

6. Timing without a clock system

6.1 CLOCK is a local transaction strobe

The signal called CLOCK is not a clock domain or a global timebase. It is one Boolean transaction signal entering one child port from its immediate owner. It need not be periodic, and no other region is required to observe it. During loading, the fixed overlay forwards CLOCK and DATA only toward the first incomplete node selected by the transaction tree. During commit and release, it propagates over that retained finite tree. It is never required to reach all cells simultaneously.

6.2 Causal size and bounded-delay model

Let

h^(R)=1+maxv∈VRdistTtx(pR,v)

be the maximum causal path measured from the owner-side neighbor, through the port ingress, to a region cell. The added one counts the physical owner-to-port forwarding stage, including for a one-cell region. This causal radius, rather than bounding-box width alone, is the relevant linear size. A serpentine tree in an L-by-L area may have h^(R)=Θ(L2). If allocation maintains a shortest-path, breadth-first, or monotone tree in a W-by-H region, then

h^(R)≤W+H−1.

Otherwise the owner uses the actual tree depth.

Let ΛH(R) be a static upper bound on every complete causal dependency which must close during a high phase. This includes an ordinary loading rise reaching the selected receiver and, on the commit cycle, commit fan-out, local copies, and acknowledgement returning to the root. It also upper-bounds the amount by which unequal rise and fall paths can shorten the high pulse observed at any participating receiver.

Let ΛL(R) be a static upper bound on every complete causal dependency which must close during a low phase. This includes falling-edge propagation, FULL changes, completion returning far enough to select the next frontier, new DATA reaching that frontier, and the final postorder release. It also upper-bounds the amount by which unequal fall and rise paths can shorten the low pulse observed at any participating receiver.

These are whole-path bounds, not single-hop delays. They are obtained from the fixed cell netlist and the finite retained transaction tree. Because one protocol phase traverses only a fixed finite number of tree paths, fixed implementation constants aH,aL,bH,bL and a worst-case tree-level delay Δ exist such that

ΛH(R)≤aHh^(R)Δ+bH,ΛL(R)≤aLh^(R)Δ+bL.

The constants are properties of the concrete loader and must be established by its gate-level static timing analysis; they are not guessed from geometric dimensions.

Let tsu, th, tpw,H, and tpw,L be the required receiver setup time, hold time, high pulse width, and low pulse width. Let m>0 be engineering margin. Delays may differ among cells and transitions but must remain within the declared whole-path bounds. No open-loop bounded-delay proof is possible if propagation or metastability is unbounded.

6.3 Owner phase contract

Define the conservative required high and low guards by

Hreq=ΛH(R)+max(th,tpw,H)+m,

and

Lreq=ΛL(R)+max(tsu,tpw,L)+m.

Let HR and LR be the actual finite source phase lengths chosen by the owner. Correctness requires

HR≥Hreq,LR≥Lreq.

The owner uses the following protocol for each bit:

  1. While CLOCK is low, present the next DATA value.
  2. Keep DATA and CLOCK low for at least LR.
  3. Raise CLOCK and keep DATA unchanged.
  4. Keep CLOCK high for at least HR.
  5. Lower CLOCK, then advance DATA only as step 1 of the next low phase.

The high bound ensures that the last high-phase consequence has occurred and receiver hold and pulse-width requirements are met before the source can lower CLOCK or change DATA. The low bound ensures that completion, rerouting, release where applicable, and the newly presented DATA have settled with setup margin before the next rise. No assumption about equal path delays or a guessed forward/return coefficient is required.

There is no single positive fixed configuration rate valid for unbounded causal radius. For every finite region, however, h^(R), HR, and LR are finite. A periodic description may write

fcfg≤1HR+LR,

but periodicity is not required.

6.4 Timing theorem and finite completion bound

Theorem 2 — finite-region timing closure. Assume a finite valid transaction tree, empty initial loader-progress bits, one owner, a quiescent application boundary, retained transaction routing, and the bounded-delay owner phase contract. Then the first 7N rising strobes, where N=|VR|, are captured by exactly one cell each in deterministic postorder, seven consecutive bits per cell. One additional guarded cycle commits the shadow records. The transaction completes and releases in finite time without a global clock, synchronized local clocks, a global or local H-tree, a continuously running clock-distribution network, a crystal oscillator, GPS, or any external time reference. The finite transaction strobe is deliberately routed over the retained tree and is permitted to arrive with bounded skew.

Proof. Lemma 1 gives a unique finite postorder at the abstract protocol level. The high and low inequalities dominate, by definition, every whole causal dependency from one source transition which can affect interpretation of the next. Induction over the finite source-transition sequence therefore makes the physical bounded-delay execution observationally equivalent to that serialized traversal. After 7N rising strobes every shadow record is full. Lemma 3 and the guarded final high and low phases commit, acknowledge, and release the retained tree even when newly installed roles or parent bits differ. No step requires simultaneous switching or a shared phase outside this one finite transaction. ∎

If the owner chooses constant finite phase lengths HR and LR for the transaction and an actual finite initial settling interval Linit, the duration of the source sequence through release is

Tsource=(7N+1)(HR+LR)+Linit.

The initial guard must satisfy

Linit≥Λinit(R)+tsu+m,

where Λinit(R) is the static bound from initial DATA presentation to stability at the first selected receiver. The equality describes the source sequence for the chosen guards; it is not an upper bound on an owner which is permitted to wait longer. When the owner or its delay replicas also have characterized finite maximum delays, substituting those maxima gives a physical worst-case completion bound. In either case every finite region has a finite witness schedule.

6.5 Producing the guard inside the owner

The owner need not measure the required guards with a crystal. It can contain two matched delay replicas constructed from the same Boolean and wire cells. If one replica stage has a characterized minimum delay δrep,min, choose finite lengths nH,nL such that

nHδrep,min≥Hreq,nLδrep,min≥Lreq.

A Boolean phase controller raises CLOCK, launches a token through the high-phase replica, and lowers CLOCK only when the delayed token returns. On lowering CLOCK it advances DATA, launches a token through the low-phase replica, and raises CLOCK only when that token returns. This is a bundled-delay sequencer, not a free-running time reference or a balanced clock H-tree. Minimum replica delay is compared against maximum protected-path delay with process, voltage, temperature, loading, and skew margin. Internal commit and release acknowledgements remain part of the finite loader protocol. The owner may observe root COMPLETE for fault detection or adaptive speed, but bounded-delay correctness does not require COMPLETE to determine the nominal phase length.

For a larger finite child, the owner instantiates a longer finite delay replica or otherwise emits wider finite phases. The initial event which starts the owner is ordinary Boolean input or retained initialization; the model does not require a quiescent circuit to begin spontaneously.

The browser implementation's guarded sequence is a direct emulator instance of this discipline: one framebuffer update advances local propagation by one neighboring step, while the owner changes the modeled configuration strobe only after a fixed number of propagation updates. Those updates emulate closure time; they are not an architectural global clock.

7. Programming and ordinary machines

7.1 Programming is installation

A program for the fabric is a finite description which an owner converts into configuration records and local application data. It may originate outside the machine as initial input, just as a bitstream, machine-code image, or Turing-machine tape may be supplied to another model. Once a suitable generator or processor is installed, further configuration streams can be computed inside the fabric.

There is no ontological boundary between program, configuration, and application data. They are distinguished by where Boolean bits are routed and which Boolean latches they control.

7.2 Finite Boolean circuits

Theorem 3 — finite-circuit embedding. Every finite Boolean circuit has a finite Cartilage configuration with the same Boolean input-output behavior after propagation settles.

Proof. Replace every gate by a finite multiplexer-and-constant network using Shannon expansion. Embed the resulting finite directed netlist into a finite rectangular portion of the grid. Use wire roles for routed segments, crossing roles where two independent routes cross, and multiplexer roles for logic. Fan-out is obtained by presenting one cell output to more than one routed successor. Because the source netlist is finite, the replacement and its planar routed embedding are finite. The configured network therefore produces the same Boolean function after the finite propagation depth settles. ∎

7.3 Sequential circuits and stored-program processors

Theorem 4 — finite transition-system embedding. Every finite deterministic Boolean transition system

qn+1=F(qn,xn)

has a finite Cartilage configuration whose state observed after each declared local publish event is exactly qn, under a compatible finite-delay discipline.

Proof. Theorem 3 embeds the finite combinational function F. Use two finite banks of the retained elements of Section 3.2 and one retained phase bit. At any time exactly one bank is active and is the sole state input to F; the other bank is inactive. During an evaluate phase, hold xn and the active bank fixed, allow F(qn,xn) to settle, and load only the inactive bank. After its setup and hold requirements are satisfied, close the inactive bank. During a nonoverlapping publish phase, change the phase bit so that the completed inactive bank becomes active. Keep the former active bank unchanged until that selector transition and all externally visible outputs have settled; it may then serve as the inactive bank for the next transition.

The combinational network never reads the bank currently being written, so transparent-latch race-through cannot occur. The evaluate guard exceeds the maximum path from active state and input through F into the inactive bank; the publish guard exceeds the maximum phase-selector and output path. These are finite static timing bounds for the finite configuration. A fence makes observations only after the publish guard, so physical bit and selector skew does not assert mathematical simultaneity. By induction on publish events, the selected bank contains exactly qn and the next selected bank contains exactly F(qn,xn). The required evaluate, close, publish, and return sequence is generated by a fixed finite phase controller built directly from the MUX latches of Section 3.2; acknowledgements or bundled-delay guards prevent it from advancing early. This controller is finite Boolean circuitry, not an invocation of the theorem being proved. It need not be a global clock and need not be shared with any other region. ∎

Registers, finite memories, arithmetic units, instruction decoders, program counters, buses, and control units have finite Boolean transition-system realizations. Therefore, for every finite memory capacity n, a conventional von Neumann computer Vn has a finite Cartilage configuration.

The construction is uniform. Given a finite Boolean or RTL description and a finite memory bound, Boolean decomposition is mechanical and a finite placement-and-routing search can enumerate successively larger finite rectangles until it finds an embedding. Efficiency is a separate engineering question; existence does not depend on noncomputable knowledge of the final layout.

Corollary — in-fabric owner. A finite controller which reads a finite configuration description and emits the DATA waveform and guarded CLOCK phases of Section 6 has a finite Boolean transition-system realization. By Theorem 4 it has a finite Cartilage configuration. The Boolean-source premise of Theorem 1 is therefore realizable inside the same fabric rather than by an external interpreter.

Corollary — finite recursive composition. For every finite nesting depth d, a Cartilage region can configure a child hierarchy of depth d using no machinery outside the fabric.

Proof. Depth zero contains no child transaction. Assume a hierarchy of depth d is internally configurable. Theorem 1 allows its owner to install another port, its local DATA/CLOCK driver wiring, and the finite owner controller of the preceding corollary, satisfying the component-acyclicity condition of Section 5.2. That child can configure a depth-d hierarchy, producing depth d+1. ∎

A conventional compiler may produce the instruction bits stored in Vn. This does not introduce a new computational primitive. The compiler is a method of producing Boolean input. It may itself be implemented as another finite or extensible Boolean machine.

The useful distinction is therefore not "Turing machine versus von Neumann machine versus Boolean fabric." A Turing machine is a convenient proof object; a von Neumann processor is a convenient programmable configuration; both are organizations of Boolean state and switching.

7.4 Potentially unbounded storage

For the abstract universality theorem, every site outside the finite seed begins in the same computable unclaimed state Av=0. An unclaimed cell ignores application and configuration traffic except a fixed local CLAIM interface.

A finite boundary controller annexes one adjacent site by the following Boolean protocol:

  1. One claimed boundary cell asserts CLAIM toward an adjacent unclaimed site.
  2. If more than one neighbor requests simultaneously, the unclaimed cell uses one fixed direction priority; well-formed allocators normally avoid the contention.
  3. The selected cell sets Av=1, sets its parent direction toward the requester, retains a quiescent installed role, and asserts ACK on that edge.
  4. After ACK, the requester treats the new cell as an owned leaf and may include it in a finite configuration transaction.

CLAIM, priority, the allocated latch, parent installation, and ACK are fixed finite Boolean circuitry. No operation names the cell by a global coordinate and no oracle chooses the eventual active size.

For universal computation, no one finite active machine is claimed to contain infinite memory. The assumption is potential extension: every finite claimed region has quiescent neighboring sites in the computably blank carrier, and the finite CLAIM protocol may be repeated without a predetermined bound.

An unbounded tape can be represented by a locally linked path of finite symbol cells. When the simulated head reaches the current end, the allocation controller claims one adjacent quiescent cell and installs its blank-symbol role. At every finite event prefix, only finitely many cells have been claimed and only a finite causal region is involved.

A sequence of finite von Neumann configurations V1,V2,… can also be generated uniformly, while a running machine can organize newly claimed blocks as linked or segmented storage. The word "heap" in this context means an application-level graph of dynamically allocated storage, not a sorted binary heap and not the ownership tree itself.

A finite physical device lacks the abstract lattice's unclaimed reservoir. Attaching another physical cell or module is therefore an explicit resource action by the environment. Once the standard local interface exists, CLAIM and all subsequent allocation/configuration behavior are Boolean; the mechanical act of supplying matter is a resource premise, not a computation performed by the existing finite circuit.

7.5 Computability theorem

A function computation begins from a finite input and the computable quiescent background. It terminates when a designated Boolean HALT output makes its declared transition; the finite Boolean values at designated output ports are the result. Exact analog transition times are not output data.

An admissible configuration has constructive execution semantics of one of two forms. In the delay-robust form, all legal delay assignments satisfying the timing contract agree on termination as well as result: for each input, either every legal execution diverges, or every legal execution eventually asserts HALT with the same finite output. In the scheduled form, every machine-visible race or arbitration is resolved by one explicitly specified computable fair scheduler, and all timing bounds needed to generate that execution are computably specified. The computed partial function is then defined by that scheduled execution. In either form, the finite machine description supplies at least one computable fair legal event schedule; in the delay-robust form that schedule is only a simulation witness, not a source of the result. Metastable amplitudes, noncomputable real delays, relative-delay races, and exact-time measurements are not oracles available to the program.

Theorem 5 — Turing equivalence under constructive premises. Cartilage with finitely described initial conditions, computable local cell circuitry, non-Zeno finite causal propagation, constructive execution semantics, and potentially unbounded finite extension computes exactly the partial recursive functions.

Proof, lower bound. By Theorem 4, finite sequential control is implementable. A locally linked path implements tape symbols and head position. Potential extension supplies another blank symbol cell whenever required. Therefore Cartilage simulates any ordinary Turing machine.

Proof, upper bound. The state alphabet of one cell is finite and its local transition is computable. From a finite or computably generated initial description, a Turing machine can enumerate the finite causal events affecting any finite event prefix under the specified computable fair scheduler. Locality, quiescence outside the finitely reached cone, and non-Zeno propagation make each such prefix finite. In the scheduled case, simulate the declared scheduler directly. In the delay-robust case, choose any fixed computable fair legal scheduler satisfying the stated bounds; by admissibility, its halting behavior and output equal those of every other legal execution. CLAIM and configuration are computable local protocols and add only finitely described active cells. HALT and output are finite Boolean observations, not measurements of arbitrary real time. Thus the simulator halts exactly on the Cartilage computation's domain and returns the same finite output. Therefore every Cartilage-computable partial function is partial recursive. ∎

An arbitrary noncomputable infinite initial pattern is excluded from the theorem. If such a pattern is supplied and made queryable, it acts as an oracle supplied in the initial condition; it is not power generated by the Boolean substrate. Ordinary quantum computation likewise changes complexity bounds, not this computability classification.

8. What classical machine distinctions become

The substrate does not erase useful distinctions between machines. It locates those distinctions in organization and resource discipline rather than in different logical primitives.

  • A fixed finite active region is a finite-state sequential machine.
  • A finite controller with one unbounded last-in-first-out path realizes a pushdown discipline.
  • A finite controller with an extensible symbol path and movable head realizes a Turing tape.
  • A processor configuration with finite memory realizes a finite von Neumann machine.
  • A uniformly extensible collection of linked storage blocks realizes potentially unbounded general storage.

A pushdown automaton remains weaker than a Turing machine because its access discipline is restricted, not because its stack uses a different kind of logic. Unbounded storage alone does not imply universality. Universality follows here because each newly available cell contributes configurable logic, routing, and retained state, and because the explicit tape construction organizes those resources into a universal machine.

9. Compactness claims

The role and ownership record admits a precise conditional minimality statement.

There are 32 directly selectable roles, so a fixed-length direct role code requires at least

⌈log232⌉=5

bits. There are four parent directions, so their direct encoding requires at least

⌈log24⌉=2

bits. If role and parent are independently selectable, there are 32·4=128 installed records, requiring at least seven bits. The base Cartilage record attains this bound.

This does not prove that Cartilage is the smallest possible universal cellular substrate, that its complete loader uses the fewest possible latches, or that two dimensions and a direct crossing role are necessary. It proves only that seven bits are optimal for this chosen direct alphabet and four-direction ownership encoding.

10. Finite realization and the emulator

A physical Cartilage device is finite. It can be tiled with other devices or extended at an exposed boundary. The theoretical unboundedness assumption is the absence of a fixed maximum over all such finite extensions, exactly as an abstract Turing tape represents the absence of a fixed tape bound.

The browser/GPGPU implementation stores each logical cell in finite packed Boolean state and repeatedly applies the fixed local cell transition. Its full-texture passes provide simulation progress and emulate nonzero propagation time. They are not visible to a configured Cartilage application as a global clock. Configuration CLOCK and DATA are ordinary modeled signals entering through a local port and moving through the ownership tree.

The implementation is evidence for the finite cell transition, role decoding, local ownership representation, serial loader, progress flags, guarded local strobe, and nested-port mechanism. The published browser transaction demonstrates the parent-preserving case. The retained transaction-route epoch, acknowledged commit/release protocol, and quiescent-site CLAIM interface used here to prove arbitrary safe reparenting and uniform lazy allocation are requirements of the robust substrate contract; they are not yet independently implemented in that shader. A particular shader, ASIC, FPGA realization, or modular assembly is conforming to the full theorem when its observable local behavior includes those mechanisms and its physical timing obeys the declared bounds.

The public Cartilage Core repository adds a second implementation layer. Its SystemVerilog fabric has a continuous combinational application plane, locally routed configuration clocks, real local positive and negative configuration edges, and a three-bit route shadow that preserves the active path through handoff. Its self-checking Verilator testbench installs all 36 seven-bit records of one 6x6 image, then checks all four rows of an interior-MUX AND circuit. This materially strengthens the finite mechanism evidence, but it does not implement the full retained-route epoch, acknowledged commit/release, arbitrary safe reparenting, or lazy CLAIM contract used by the general theorem. The browser and RTL remain logical models, not physical timing or silicon signoff.

11. Explicit premises and nonclaims

The result depends on the following premises.

P1 — Boolean cell circuitry. Every cell's application and loader transformations are finite Boolean circuits with retained Boolean state.

P2 — Local carrier. Cells communicate only through a fixed finite number of local physical links.

P3 — Causal realization. Signal and storage transitions have characterized positive, finite behavior, including the worst-case loader bounds used in Section 6 and a non-Zeno guarantee excluding infinitely many causally successive local transitions in finite time.

P4 — Valid ownership. Every reconfigured region is one finite transaction-closed component of T(A,C,P); its retained claim, role, and direction fields decode to a rooted acyclic tree with exactly one port root. Integrity-repair mutations are suspended while that transaction is active. The cells remain claimed throughout the base transaction, and their newly installed role and direction fields jointly decode with those claim bits to a valid final forest before release. Any claimed nested hierarchy also satisfies the explicit driver-wiring and component-acyclicity condition of Section 5.2.

P5 — Owner timing. The owner separates configuration transitions by a bound calculated for the target region's finite causal depth.

P6 — Constructive initialization, execution, and observation. Initial configurations, input streams, CLAIM actions, timing generators, HALT, and output observations are finite or computably generated. The machine description supplies a computable fair legal event schedule. Machine-visible nondeterminism is either invariant in both termination and output over all legal delays, or resolved by a specified computable fair scheduler with computably specified timing bounds. Exact analog time, arbitrary real delays, race outcomes, and metastable quantities are not program-visible oracle values.

P7 — Potential extension. The theoretical carrier has a computably uniform quiescent background, and the finite Boolean CLAIM protocol may annex adjacent sites without a predetermined bound. Physical attachment is a separate resource premise for finite implementations.

P8 — Exclusive transaction. One designated owner drives a child's configuration ingress, and no overlapping transaction mutates that child before release.

The paper does not claim:

  • that Boolean equations derive physical spacetime or energy;
  • that every arbitrary asynchronous circuit is hazard-free;
  • that partially installed application logic is useful before completion and release;
  • that a global snapshot is physically taken by the machine;
  • that unbounded passive memory alone implies universality;
  • that a noncomputable infinite pattern is generated by the substrate;
  • that Cartilage exceeds Turing computability under the constructive premises;
  • that the seven-bit installed record is a global minimum for universal computation;
  • that parent pointers alone provide cryptographic protection against a physically adversarial driver;
  • that a browser shader proves the electrical limits of a future physical implementation.

12. Conclusion

Cartilage begins with a fixed repeated Boolean cell circuit. Boolean configuration bits select each cell's effective application role. Boolean parent bits select its local ownership relation. Boolean loader state routes Boolean DATA and CLOCK signals through the resulting finite region. Boolean completion state determines when records and subtrees are finished. The installed result is another Boolean circuit, which may generate and install further Boolean configurations.

Thus the machine which is configured and the machine which performs configuration are not different ontological layers. They are two uses of the same Boolean substrate.

Time enters operationally as causal propagation of switching events, not as a universal clock variable. A finite owner waits long enough for the finite region it owns. The required interval grows with that region's finite causal depth and can be produced by a finite path in the same fabric. Nothing requires global synchronization, a global or local H-tree, a continuously running clock-distribution network, a crystal, GPS, or a non-Boolean timing controller. A finite local strobe is routed through the selected transaction tree with bounded, tolerated skew.

Finite Boolean circuits, sequential machines, conventional processors, tapes, stacks, and linked memories are configurations and resource disciplines inside this substrate. Potential extension removes a fixed resource bound; it does not introduce a new logical primitive. Under constructive initial conditions, the resulting model is Turing-equivalent.

Under the explicit carrier, causal-propagation, non-Zeno, timing, and resource premises of this paper, every machine-visible value and every computational transformation involved in application behavior, configuration transport, ownership, traversal, commit, release, and circuit replacement is Boolean. No non-Boolean computational value, transformation, interpreter, or configurator is required.
