Caching avoids recomputation by keeping intermediate values available. Reversible computing makes the other side of that bargain visible: copied information has to be accounted for, cleaned up, or uncomputed.

LinkedIn postPosted 2019-06-05.

The Copy Cost

Ordinary machines use copies constantly. A cached value can save work by preventing a branch of a computation from being recomputed. That is useful, but it is not physically neutral: the machine must allocate state, preserve it, overwrite it, or erase it.

In reversible computing, that bookkeeping becomes part of the design problem. If an intermediate result is copied for later use, the circuit also needs a path for cleaning up temporary state without throwing away information that still matters.

The Engineering Question

The useful comparison is storage versus recomputation. A saved value costs area, memory, energy, fanout, and later cleanup. Recomputing can be expensive too, but it may avoid carrying extra state through the rest of the circuit.

A reversible design has to decide where copied information belongs, how long it should live, and whether it can be uncomputed cleanly after the dependent work is finished.
